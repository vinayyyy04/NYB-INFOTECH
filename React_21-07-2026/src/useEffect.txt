# Understand the Lifecycle Behavior of useEffect

* **useEffect** is a React Hook used to perform side effects such as API calls, timers, event listeners, and DOM manipulation after a component renders.

## Lifecycle Behavior

### 1. Mount (Component is Created)

* The effect runs after the component renders for the first time.
* If the dependency array is empty (`[]`), it runs only once.

### 2. Update (Component Re-renders)

* If dependencies are provided, the effect runs again whenever any dependency changes.
* If no dependency array is provided, the effect runs after every render.

### 3. Unmount (Component is Removed)

* If the effect returns a cleanup function, React executes it before the component is removed from the DOM.
* Cleanup is commonly used to clear timers, remove event listeners, or cancel subscriptions.

## Dependency Array Behavior

* **No dependency array:** Runs after every render.
* **Empty dependency array (`[]`):** Runs only once after the initial render.
* **Dependency array (`[dependency]`):** Runs after the initial render and whenever the specified dependency changes.

## Cleanup Function

* The cleanup function runs:

  * Before the effect runs again (if dependencies change).
  * Before the component unmounts.

## Summary

* **Mount:** Effect executes after the first render.
* **Update:** Effect executes when specified dependencies change.
* **Unmount:** Cleanup function executes before the component is removed.
* **No dependencies:** Effect runs after every render.
* **Empty dependency array:** Effect runs only once.
* **With dependencies:** Effect runs only when those dependencies change.
